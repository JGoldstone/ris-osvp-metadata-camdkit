OpenTrackIO Documentation

Overview

OpenTrackIO is a free and open protocol designed by the RIS OSVP SMPTE group that seeks to improve interoperability in Virtual Production encoding, tracking and robotic systems.

This documentation is designed for those producing and consuming tracking data.

OpenTrackIO defines static and dynamic JSON samples that contain a wide range of metadata about the device, its transform(s), camera and lens.

All the fields described should be considered optional by the consumer (although for high-quality tracking see the recommended set in the samples below).

OpenTrackIO employs the OpenLensIO mathematical model for the practical application of lens distortion in Virtual Production.

Static Sample

It is recommended that a static metadata sample is transmitted approximately every 2 seconds. It describes the metadata context of the dynamic samples that follow, with data that may change - for example - every take, but will not change every frame.

Dynamic Sample

It is recommended that dynamic metadata samples are transmitted every frame (i.e. to coincide with the video frames from a camera). It provides a snapshot of the status of the tracking system at that instant.
